## Claims From Source

- tooday should be public and open source.
- The default direction is local/self-hosted, not subscription-based.
- Users should be able to deploy it on their own server and pull updates from
  GitHub.
- AI should be BYOK in a later layer.
- A local Codex account connection is a possible future integration.

## Implications

- README and docs must avoid implying a hosted SaaS dependency.
- AI features should be optional and should not block core planner workflows.
- Self-hosting docs should clearly separate current browser-local storage from
  future cross-device sync.
